Git + Gitee（码云）使用手册

发表于 2021-10-08 分类于 工具

重点：

  1. 了解Git、Gitee的概念；

  2. 安装Git、注册Gitee；

  3. 尝试创建Gitee的远程仓库，并与本地仓库进行关联；

  4. 掌握分支概念，掌握如何进行本地仓库与远程仓库同步。

Git简介 & Gitee简介

Git

什么是Git：Git是一个分布式版本控制系统，简单来说就是一个代码管理工具，方便团队开发时，对于代码的统一管理。

Gitee

什么是Gitee：Gitee是国内的一家软件项目托管平台，国外也有类似的平台GitHub，但是GitHub在国内访问并不稳定。Gitee可以看作是软件项目的“云盘”，可以将搭配Git，进行软件项目代码的管理。

Git安装 & Gitee注册

Git安装

官网下载地址：http://git-scm.com/downloads

国内镜像下载地址：https://npm.taobao.org/mirrors/git-for-windows/

Gitee注册

Gitee官网：https://gitee.com/，注册很简单，不多说。

在本地Git登录Gitee帐号

配置用户名 & 邮箱

相当于登录。

在桌面右键，点击Git Bash Here，打开Git的命令行窗口。

输入 git config --global user.name "自己的用户名"，回车。

输入git config --global user.email "自己的邮箱"，回车。

用户名和邮箱就配置完了。

配置SSH Key

为什么配置SSH Key：配置SSH Key的目的是实现免密登录，否则每次提交代码时都需要输入邮箱密码，比较繁琐。

在命令行窗口输入命令ssh-keygen -t rsa -C "自己的邮箱"生成密钥文件，会有3次提示，直接回车就行。如果密钥文件之前已经存在，则会提示是否覆盖，覆盖即可。

在标红的目录下可以找到生成的密钥文件：

记事本打开id_rsa.pub，即为公钥，全选、复制其中的内容。

登录Gitee，进入个人主页–>个人设置–>SSH公钥，将刚才复制的公钥粘贴上去，确定。

返回命令行窗口，输入命令ssh -T git@git.oschina.net，弹出提示，输入yes。

出现successfully即可。

Git：一般使用

Git分区

Git分为工作区、暂存区、本地仓库、远程仓库等分区，以一个代码文件为例，其在Git中的路线是：工作区修改代码–>暂存区暂时存放（此时修改未保存到本地仓库）–>提交到本地仓库–>推送到远程仓库。

初始化本地仓库

git init：将当前文件夹初始化为一个本地仓库。

查看仓库状态

git status：查看当前仓库状态。

添加文件

git add <file>：将指定的文件添加到暂存区。

git add -A：将所有文件添加到暂存区。

提交

git commit -m "提交信息"：将暂存区的所有文件提交到本地仓库，每次提交时都需要附加提交信息（注释）。

查看文件修改和撤销修改

git diff：查看文件有哪些修改。

git checkout <file>：未add，撤销修改。

git reset HEAD <file>：已add未commit，撤销修改。

git log：查看提交记录，可以查看每个版本的哈希码。

git reset --hard 哈希码前七位：已commit，撤销修改（版本回退）。

本地仓库与远程仓库同步

分支

什么是分支：我认为分支就像项目的“平行宇宙”，一个项目可以有不同的分支，不同的角色可以在不同的分支上对代码进行修改，但彼此之间不会相互影响，就像“平行宇宙”一样。其中主分支一般称为master。

至于为什么划分分支管理，可以自行搜索一下。

切换分支

git branch <branchname>：创建分支。

git checkout <branchname>：切换分支。

git checkout -b <branchname>：创建并切换分支。

git branch -d <branchname>：删除分支 。

拉取

git pull <远程主机名> <本地分支名>:<远程分支名>：通用命令格式。

git pull origin master:<branchname>：将远程仓库的master分支内容拉取并与分支合并。

git pull origin master：将远程仓库的master分支内容拉取并与当前分支合并。

推送

git push <远程主机名> <本地分支名>:<远程分支名>：通用命令格式。

git push origin master:master：将本地的master分支推送到远程仓库的master分支。

git push --force origin master：本地版本与远程版本有差异，强制推送。

合并分支

git merge <branchname>：合并分支到当前分支。

案例

一般提交流程：先add，再commit，再push。

  1. Gitee创建远程仓库；
  2. 将远程仓库克隆到Git本地仓库；
  3. 本地仓库新建分支（以名字首拼命名，如xsh），并在该分支下新建空文件test.txt，最后将修改提交到远程仓库的master分支。
  4. 远程仓库新建分支（同样以名字首拼命名，如xsh），并将本地仓库同名分支的文件提交到此分支中。
  5. 将远程仓库的新建分支（如xsh），拉取到本地仓库的master分支。
