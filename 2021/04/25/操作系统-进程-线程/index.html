发表于 2021-04-25 分类于 操作系统

进程状态转换

进程与线程

区别与联系：

  1. 单位：进程是操作系统分配资源的最小单位，线程是CPU进行任务调度的最小单位。一个进程可以包含多个线程，所以进程和线程都是CPU工作时间段的描述，只是颗粒大小不同。
  2. 数据共享：不同进程之间的数据很难共享，同一进程下不同线程之间的数据很容易共享。
  3. 代码和数据空间：每个进程都有独立的代码和数据空间，同一类线程共享代码和数据空间，但是每个线程都有自己独立的运行栈和程序计数器；进程比线程要消耗更多的计算机资源，线程可以看作轻量级的线程，因此线程之间切换开销小。
  4. 互相之间的影响：进程之间不会相互影响，但一个线程挂掉会导致整个进程挂掉。
  5. 内存分配：操作系统在运行时为每个进程分配不同的内存空间，但对于线程，操作系统不会为其分配除CPU外的资源，线程之间只能共享资源，因此线程之间共享内存。
比较方向 进程 线程
单位 资源分配的最小单位 任务调度的最小单位
数据共享 进程之间数据不共享 同一进程下，线程之间数据共享
代码和数据空间 进程有独立的代码和数据空间 同一进程下，线程之间共享代码和数据空间，但每个线程都有自己的运行栈和程序计数器。
相互影响 进程之间互不影响 同一进程下，一个线程挂掉会导致整个进程挂掉
内存分配 操作系统分配内存 同一进程下，线程之间共享内存

线程越多越好吗

线程是越多越好吗？

不是的，也分情况。

假如业务逻辑都是计算型（CPU密集型）的，不涉及到 IO，且只有一个CPU核心，那么只有一个线程是最好的，线程多了就需要增加线程切换的计算，那么CPU就不能将计算资源完全用于业务上，此时线程越多会造成CPU利用率下降。

假如业务逻辑不是CPU密集型，如Web场景，此时业务逻辑属于 IO密集型，当一个线程在等待 IO 资源时，完全可以将CPU计算资源让给其他线程，这样可以充分利用CPU资源。

但是线程数量也有限度，一般可以用公式：最佳线程数 = [ 任务执行时间 / (任务执行时间 - IO等待时间) ] * CPU内核数来计算。线程数高于这个数量，CPU就要进行多余的线程切换，浪费CPU计算资源；线程数低于这个数量，线程就要在CPU中进行IO等待，也无法充分利用CPU计算资源。

总之，要尽可能榨取CPU的计算能力，如果CPU处于饱和状态且没有多余的线程切换，此时CPU与线程就处于完美状态，如果再增加线程并发量，就会造成性能下降。

线程数量 导致结果
线程多了 线程切换频繁，CPU需要分出一部分计算资源用于线程切换，不能充分利用CPU的计算能力。
线程少了 线程在CPU中进行IO等待，CPU无法保持时刻处于运行，浪费CPU计算能力。

进程的组成

三部分：进程控制块（PCB）+程序段+数据段

进程的通信方式

方式 描述 特点
无名管道 是一种特殊的共享文件，用于具有亲缘关系的进程间的通信。无名管道是半双工的，即数据只能在一个方向上流动。无名管道不是普通的文件，不属于文件系统，只存在于内存中。 速度慢，容量有限，只能在父子进程间进行通信。
FIFO命名管道 也是一种特殊的共享文件，可以用于无关的进程间的通信。与无名管道不同，FIFO命名管道以特殊设备文件的形式存在于文件系统中。 速度慢，任何进程之间都能进行通信
消息队列 消息的链接表，存放在内核中。一个消息队列由一个标识符（队列ID）来标识。 容量受系统限制，且第一次读时要注意上一次没有读完数据的问题。
信号量 一个计数器，用于实现进程间的同步与互斥，不用于储存进程间的通信数据。 不能传递复杂消息，只能用来同步。
共享内容 给两个或多个进程指定一个存储区，一般需要配合信号量使用。 能控制容量，速度快，但需要保持同步（比如一个进程在写的时候，另一个进程要注意读写的问题，相当于线程中的线程安全）。共享内存区同样可以用作线程间通讯，不过没这个必要，线程间本来就已经共享了同一进程内的一块内存。

进程调度策略

算法 描述 特点
先来先服务（FCFS） 先请求CPU的进程先分配到CPU。 可以作为作业调度算法也可以作为进程调度算法，队列实现，非抢占。按作业或者进程到达的先后顺序依次调度，对于长作业比较有利。不会产生“饥饿”现象。
短作业优先（SJF） 从就绪队列中选择估计时间最短的作业进行处理，直到得出结果或者无法继续执行。 作业调度算法。平均等待时间最短。不利于长作业；未考虑作业的重要性；运行时间是预估的，并不靠谱。会产生“饥饿”现象。
优先级调度 优先级越高越先分配到CPU，相同优先级先到先服务。 可以是抢占的，也可以是非抢占的。低优先级进程无穷等待CPU，会导致无穷阻塞或饥饿。
时间片轮转 按到达的先后对进程放入队列中，然后给队首进程分配CPU时间片，时间片用完之后计时器发出中断，暂停当前进程并将其放到队列尾部，循环。 可抢占的。队列中没有进程被分配超过一个时间片的CPU时间，除非它是唯一可运行的进程。如果进程的CPU区间超过了一个时间片，那么该进程就被抢占并放回就绪队列。
