Java-JVM之垃圾收集器

发表于 2021-04-27 分类于 JVM

简单介绍一下JVM中的常用垃圾回收器。

回收器概览

回收器 分代 特点 适用场景 配合使用
Serial 新生代 单线程需要STW(Stop-The-World) Client模式 Serial Old、CMS
ParNew 新生代 Serial的多线程版本需要STW减少了STW时间，提升应用响应速度 Server模式 Serial Old、CMS
Parallel Scavenge 新生代 多线程使用复制算法注重吞吐量 适合做后台运算等吞吐量大、不需要太多交互的任务 Serial Old、Parallel Old
Serial Old 老年代 单线程需要STW 主要在Client模式下搭配Serial回收器使用，也可以在Server模式下作为CMS的后备预案。 Serial、ParNew、Parallel Scavenge
Parallel Old 老年代 Parallel Scavenge的多线程版本使用标记整理法注重吞吐量 适合吞吐量大的任务 Parallel Scavenge
CMS 老年代 使用标记清除法注重最短STW时间总体上看，可以认为回收线程与用户线程是并发执行的 注重服务响应速度 Serial、ParNew
G1 / 使用标记整理法能与应用程序线程并发执行能建立可预测的停顿模型，能在用户指定的时间进行STW。 面向服务端 /

Serail与ParNew共用了相当多代码，CMS与前两者共用同一套代码框架；Parallel Scavenge与G1则没有使用与它们相同的GC收集器代码框架，而是后来另起炉灶。

从JDK 9开始，G1成为了默认的垃圾回收器。

CMS与G1

CMS执行过程

前面概览已知，CMS使用的是标记清除法，但实际上需要进行多次标记。

  1. 初始标记（STW initial mark）：暂停虚拟机，从GC Roots开始扫描并标记，且只扫描标记能与GC Roots直接关联的对象，所以STW的时间较短。
  2. 并发标记（Concurrent marking）：在初始标记的基础上继续向下追溯标记，此阶段应用程序线程与并发标记线程并发执行，用户不会感受到停顿。
  3. 并发预清理（Concurrent precleaning）：扫描标记在并发标记阶段进入老年代的对象，此阶段仍然是并发的。此阶段目的是减少下一阶段的工作量，因为“重新标记”会STW。
  4. 重新标记（SWT remark）：暂停虚拟机，扫描CMS堆中剩余对象，从GC Roots开始向下追溯，并处理关联对象。
  5. 并发清除（Conccurent sweeping）：清理垃圾对象，此阶段收集器线程与应用程序线程并发执行。
  6. 并发重置（Concurrent reset）：重置CMS收集器的数据结构状态，等待下一次垃圾回收。

G1执行过程

G1的执行过程与CMS类似

  1. 初始标记：暂停虚拟机，进行初始标记，并且会捎带触发一次young GC。
  2. 并发标记：在初始标记的基础上继续向下追溯标记，并发标记线程与应用程序线程并发执行，且有可能被young GC打断。并发标记在整个堆中进行（不分代，分区），如果发现某区域中所有对象都是垃圾对象，那这个区域会立即被回收。并发标记过程中也会对每个区域的对象活性（区域中存活的对象比例）进行计算。
  3. 最终标记：补充收集并发标记阶段产生的新垃圾，使用了更快的算法：SATB。
  4. 筛选回收：暂停虚拟机，选择活性低的区域（同时考虑停顿时间），等待下次young GC一起收集。
  5. 回收完成：新的young GC清理被计算好的区域。回收完成后，有些区域可能还存在垃圾对象，这可能是因为这些区域中对象活性较高，回收不划算，也可能是因为迎合用户设置的停顿时间，不得不舍弃一些区域的收集。

CMS与G1的比较

比较方面 CMS G1
设计目标 CMS收集器的设计目标是获取最短回收停顿时间（STW时间），只有在初始标记与重新标记时会产生STW。 G1收集器面向服务端，在满足短时间停顿时，还能达到较高的吞吐量。
GC算法 基于标记清除法，清理过程会产生大量的空间碎片。 堆比较大时可以采用复制算法，碎片化问题不严重；整体上看属于标记整理法，局部（Region之间）属于复制算法。
CPU资源利用 CMS收集器对CPU资源敏感，在并发阶段，虽然不会导致用户线程停顿，但会占用CPU资源导致应用程序变慢，吞吐量下降。 G1收集器适用于多核CPU、大内存容量的服务端系统，其能充分利用CPU、多核环境下的硬件优势，使用多个CPU或多核CPU来缩短STW时间。
内存划分 堆中的老年代 堆不分代，分成多个块（Region），块各自进行回收。
适用场景 CMS适用于小内存应用。 G1收集器适用于多核CPU、大内存容量的应用。
