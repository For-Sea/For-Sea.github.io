Ajax的简单使用与介绍

Ajax概述

Ajax即Asynchronous Javascript And XML（异步JavaScript和XML）在 2005年被Jesse James Garrett提出的新术语，用来描述一种使用现有技术集合的‘新’方法，包括: HTML 或 XHTML, CSS, JavaScript, DOM, XML, XSLT, 以及最重要的XMLHttpRequest。 使用Ajax技术网页应用能够快速地将增量更新呈现在用户界面上，而不需要重载（刷新）整个页面，这使得程序能够更快地回应用户的操作。

Ajax 不是新的编程语言，而是一种使用现有标准的新方法。

用了Ajax技术可以不刷新或跳转整个页面，即可加载部分网页内容。

Ajax原理

Ajax的原理流程如下：

创建XMLHttpRequest–>设置响应函数和访问页面–>执行访问–>服务器返回访问页面的响应–>响应函数处理服务器的响应–>渲染页面

Ajax例子分析

  1. 创建XHR对象即XMLHttpRequest对象，它就像一个线程悄悄运行，与服务器进行交互。

  2. 设置并发出请求

    xmlhttp.open(“GET”, url, true);

    xmlhttp.send(null);

    XHR对象使用open()函数设置要访问的页面（URL），使用send()函数执行访问。

    方法 描述
    open(method,url,async) |规定请求的类型、URL 以及是否异步处理请求。 method：请求的类型；GET 或 POST
    url：文件在服务器上的位置
    async：true（异步）或 false（同步）
    send(string) 将请求发送到服务器。
    string：仅用于 POST 请求

  3. 设置响应函数

    xmlhttp.onreadystatechange = checkResult;

    function checkResult() {

    ​ document.getElementById(‘info’).innerHTML = xmlhttp.responseText;

    ​ }

    Ajax访问页面后，服务端会发送响应回来，此时Ajax就需要设置函数对响应进行处理，checkResult()就是响应函数。

  4. 处理响应信息

    checkResult()函数对响应的信息进行处理，其中：

    xmlhttp.readyState 4 表示请求已完成。

    xmlhttp.status 200 表示响应成功。

    xmlhttp.responseText; 用于获取服务端传回来的文本，响应为XML时使用属性responseXML，此外都使用responseText。

    document.getElementById(‘checkResult’).innerHTML 设置span的内容为服务端传递回来的文本。

    属性 描述
    onreadystatechange 存储函数（或函数名），每当 readyState 属性改变时，就会调用该函数。
    readyState 存有 XMLHttpRequest 的状态。从 0 到 4 发生变化。
    0: 请求未初始化
    1: 服务器连接已建立
    2: 请求已接收
    3: 请求处理中
    4: 请求已完成，且响应已就绪
    status 200: “OK”
    404: 未找到页面
