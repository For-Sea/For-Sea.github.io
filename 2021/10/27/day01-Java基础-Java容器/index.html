Java基础

面向对象

面向对象与面向过程的区别

面向过程：自顶向下，逐步求精。将过程分解成各个函数（模块），再依次调用执行。简单高效，执行快速。

面向对象：模拟现实世界，针对数据，将数据与其操作抽取并封装成对象。易扩展、易复用、低耦合。

封装

什么是封装：将数据与对数据的操作封装成类。

封装有什么好处：隐藏信息、减少耦合、便于复用、方便扩展、提高安全性。

继承

什么是继承：子类继承父类的特征和行为。

继承有什么好处：提高代码复用性，便于对父类进行扩展。

注意点：

  • super、this关键字：super指向父类，this指向自己。

  • final关键字：修饰的类不能被继承，修饰的方法不能被重写。

  • 构造器：子类不能继承父类的构造器，只能调用父类构造器；若父类含有参构造器，则子类必须显式调用该有参构造器；若父类不含有参构造器，则子类隐式调用无参构造器。（PS：为什么写了有参构造器，要再写一个无参构造器。）

多态

什么是多态：同一个行为，拥有不同的表现形式或形态。

多态有什么好处：应对需求变化，提高代码可扩展性、灵活性。

  1. 多态在Java中的表现形式（使用场景）：
    1. 前提：编译、重写、父类引用指向子类对象 List<String> list = new ArrayList<>();。
    2. 处理：成员变量、静态方法看左边；非静态方法：编译看左边，运行看右边。
    3. 表现：方法重写、接口、抽象类与抽象方法、统一传参类型。
  2. 静态绑定与动态绑定：
    1. 什么是绑定：在多态的情况下，运行时确定调用的是父类还是子类的变量或方法，这个过程叫绑定。绑定分为静态绑定和动态绑定。
    2. 静态绑定（早期绑定）：发生在编译时期，使用private、static、final 修饰的变量或方法使用静态绑定；重载使用静态绑定完成。
    3. 动态绑定（后期绑定）：发生在运行时期，虚方法（Java中为能被重写的普通方法）使用动态绑定；重写使用动态绑定。
    4. 了解绑定有什么好处：使用动态绑定需要在运行时确定使用父类还是子类的变量或方法，比静态绑定耗时，因此在不影响整体设计的前提下，可以考虑用private、static、final进行修饰。
    5. 参考链接：https://droidyue.com/blog/2014/12/28/static-biding-and-dynamic-binding-in-java/
  3. 向上转型与向下转型：
    1. 什么是向上转型：父类引用指向子类对象。
    2. 什么是向下转型：在向上转型的前提下，将父类引用更改为子类引用。
    3. 向上转型有什么好处：传递参数时，使参数能够统一（只需要将子类向上转型，然后传递子类的公共父类作为参数即可）。
    4. 向下转型有不安全的情况（原对象是父类对象），可以使用instanceof关键字保证安全性。boolean result = a instanceof ClassA

延申：面向接口编程

什么是面向接口编程：将代码的定义与实现分离，表现为先定义接口，再定义接口的实现类。

面向接口编程有什么好处：更加抽象，更加面向对象；提高编程的灵活性；实现高内聚、低耦合，提高可维护性

延申：面向对象的设计原则

六大原则：

  1. 单一职责原则：一个类只实现一个功能。封装

  2. 开闭原则：对扩展开发，对修改关闭。封装

  3. 里氏替换原则：有父类存在的地方，将其替换成子类也不会产生错误。继承

  4. 依赖倒置原则：模块间的依赖通过抽象发生，实现类之间不发生直接的依赖关系，其依赖关系是通过 接口或抽象类产生的。即依赖抽象，而不依赖具体的实现。多态

  5. 接口隔离原则：客户端不应该依赖其不需要的接口。多态

  6. 迪米特原则：一个类应该对自己调用的类知道得最少。封装

泛型

泛型

什么是泛型：将数据类型参数化。可以用于定义泛型方法或泛型类。

泛型有什么好处：类型安全，过滤出类型不符的异常；消除强制类型转换；提高代码可读性。

泛型的底层原理：类型擦除

什么是类型擦除：使用泛型时，给定的数据类型会在编译器编译时去掉，替换成原始类型（Object或指定的类型边界），这个过程称为类型擦除。（如在代码中定义List<Object>和List<String>等类型，在编译后都会变成List，JVM看到的只是List，而由泛型附加的类型信息对JVM是看不到的。）

类型擦除参考链接：https://www.cnblogs.com/wuqinglong/p/9456193.html

为什么要使用类型擦除：https://www.pulpcode.cn/2017/12/30/why-java-generic-use-type-eraser/，主要是为了向后兼容。

类型擦除的问题及解决方案：

  1. 既然有类型擦除，那么如何实现数据类型过滤？答：Java编译器先检查代码中泛型的类型，再进行类型擦除，再进行编译。

  2. 既然有类型擦除，那么为什么返回值还是指定的数据类型呢？ 答：返回时根据指定的数据类型进行了强制转换。

  3. 类型擦除与多态的冲突：桥方法。

反射

动态语言与静态语言

什么是动态语言和静态语言：动态语言是指在运行期间，代码可以根据某些条件改变自身结构的语言，如Python、JavaScript；静态语言则是在运行时不能改变代码自身结构的语言，如Java、C++。

反射

Java虽然是静态语言，但是可以通过反射达到部分动态语言的效果。

什么是反射：可以在加载一个在运行时才得知名称的类，获得其完整构造，并生成其对象实体、或对其变量复制、或调用其方法。

反射有什么好处：利用配置文件提高灵活性，使外部调用更方便。缺点是性能低，没有直接new创建对象快。

反射的使用场景：Spring的IOC实现；JDBC加载数据库驱动。

反射的使用：

修饰符

访问修饰符

  • default：同一包内可见。

  • private：同一类内可见。

  • protected：同一包内及所有子类可见。

  • public：对所有类可见。

static关键字

修饰变量或方法。

  • 修饰变量：无论有多少实例对象，只有一份静态变量的拷贝。
  • 修饰方法：可以在不实例化对象的情况下，直接使用方法。

final关键字

修饰变量、方法、类。

  • 修饰变量：常量，不能再赋值。

  • 修饰方法：可以被子类继承，但不能再重写。

  • 修饰类：不能被继承。

String、StringBuilder、StringBuffer底层区别

可变性

  • String底层是final类型的字符数组private final char[] value，不可变（PS：Java 9之后改为了字节数组byte[]）。
  • StringBuilder与StringBuffer继承自AbstractStringBuilder，是可变的。

线程安全性

  • String底层是不可变的，不能被修改，因此线程安全。

  • StringBuffer底层方法加了同步锁，是线程安全的。

  • StringBuilder没有加同步锁，是非线程安全的。

性能

  • StringBuilder比StringBuffer性能高一点，但存在线程不安全的风险。
  • 建议操作少量数据，使用String；单线程操作字符串，使用StringBuilder；多线程操作字符串，使用StringBuffer。

BIO、NIO、AIO

IO模型

什么是IO模型：应用程序对操作系统内核发起IO调用（系统调用），操作系统相关内核负责执行具体的IO操作。一般IO有磁盘IO（读取数据）和网络IO（请求和响应）。

常见的IO模型有哪些：

  • 同步阻塞IO

  • 同步非阻塞IO

  • IO多路复用

  • 信号驱动IO

  • 异步IO

BIO

什么是BIO：Blocking IO，同步阻塞IO。应用程序发起IO调用后，会一直阻塞，直到内核将数据拷贝到用户空间。

NIO

什么是NIO：Java 1.4 引入，Non-blocking IO或者New IO。一般认为IO多路复用（也有人认为是同步非阻塞IO）。

什么是同步非阻塞IO：应用程序轮询发起IO调用，直到内核执行IO操作，执行IO操作期间线程仍然是阻塞的。

什么是IO多路复用：应用程序发起select调用，询问内核是否准备好数据，当数据准备好后，再进行IO调用，内核执行IO操作，执行IO操作的过程依然是阻塞的。

IO多路复用中，有一个选择器（Selector）的概念，也称为多路复用器。通过Selector，一个服务端线程可以与多个客户端相连，当客户端的数据准备好后，才为该客户端进行服务。

AIO

什么是AIO：Asynchronous IO，异步IO模型。应用程序发起IO调用后直接返回，不会阻塞，当内核处理完成，操作系统会通知线程执行后续操作。

equals与hashCode

  • equals：一般情况下与”==”作用相等，若是基本数据类型，比较的是值；若是引用对象类型，比较的是地址。
  • hashCode：生成哈希码，用以确定该元素在散列表中的索引位置。一般相同元素哈希码不同，但是也有可能出现哈希冲突，即不同元素拥有相同的哈希码。

为什么重写equals方法必须重写hashCode方法？

答：向散列表存放元素时，先比较哈希码是否一致，节省比较成本，再用equals进行比较。以向HashSet中存放String类型的元素为例，其重写了equals方法，若不重写hashCode方法，则会出现字符串相等，但哈希码不相等，因此存放了重复元素的情况。

clone

  • 浅拷贝：基本数据类型直接拷贝值，引用数据类型创建新的引用指向原地址。
  • 深拷贝：基本数据类型直接拷贝值，引用数据类型开辟新的内存空间，创建一个相同的新对象和指向该对象的引用。

自动拆箱与自动装箱

自动拆箱与自动装箱

Java为每种基本数据类型提供了对应的包装类。

  • 自动装箱：自动将基本数据类型转换为包装器类型，Integer i = 10;。Integer i = new Integer(10);
  • 自动拆箱：自动将包装器类型转换为基本数据类型，int j = i;。

为什么需要自动拆箱与自动装箱：基本数据类型使用的是值传递，包装器类型使用的是引用传递，使用包装器类型可以在传递参数后对数值进行更新。

装箱和拆箱是怎样实现的：以Integer为例，装箱是调用了Integer.valueOf(int n)方法，拆箱是调用了Integer.intValue()方法，其他基本数据类型的装箱拆箱也类似。

自动拆箱与自动装箱参考链接：https://www.cnblogs.com/dolphin0520/p/3780005.html

Java集合

集合框架：数组不能满足根据不同数据结构存储数据类型，且不能扩容，引入集合框架可以满足对数据存储的不同需求。

分析集合：底层结构、插入过程、扩容过程、线程安全等方面进行分析。

List

ArrayList

  • 底层：Object[] 数组

  • 线程不安全

  • 插入删除：默认插入在末尾，如果选择在中间插入删除的话，需要移动数组元素。

  • 支持快速随机访问

LinkedList

  • 底层：双向链表

  • 线程不安全

  • 插入删除：头插尾插很方便，如果选择在中间插入删除的话，需要先移动到前一位，再进行链表的插入删除。

  • 不支持快速随机访问

Vector

  • 底层：Object[] 数组
  • 线程安全，效率低

Set

HashSet

  • 无序，唯一
  • 底层基于HashMap

TreeSet

  • 有序，唯一
  • 底层：基于红黑树实现

LinkedHashSet

  • 无序，唯一
  • 底层：HashSet的子类，底层通过LinkedHashMap实现。

Map

HashMap

  • 底层：1.8 之前，数组+链表，使用链表解决哈希冲突；1.8 开始，数组+链表+红黑树，当链表长度大于阈值（默认为8），且此时数组长度大于64，链表会被转化为红黑树，若此时数组长度不大于64，则会先选择对数组进行扩容。

  • 线程不安全

  • 支持null key和null value，null key 只能有一个，null value可以有多个

LinkedHashMap

  • 底层：继承自HashMap，底层仍然是数组+链表+红黑树，但是在此基础上添加了一条双向链表。

HashTable

  • 底层：数组+链表，数组为主体，使用链表解决哈希冲突。

  • 线程安全，加了synchronized修饰方法

  • 出于线程安全，不允许null key和null value

fali-fast、fail-safe机制

本质目的还是保证对集合的并发操作的安全性。

参考链接：https://juejin.cn/post/6941476861704667166

fail-fast

什么是fail-fast：快速失败机制，在迭代时，如果发现 该集合数据 结构被改变 (modCount != expectedModCount)，就会 抛出 ConcurrentModificationException。是Java集合中的一种错误检测机制。单线程或多线程条件下，创建迭代器Iterator对集合进行遍历时，除了迭代器本身的方法（如remove()）可以对集合结构进行改变（如添加或删除）外，其他因素如果导致集合结构发生改变，就可能会抛出异常（ConcurrentModificationException）。

fail-fast有什么好处：增强数据安全性。当多个线程对同一个集合进行高强度增删改查时，可能会出现数据异常。

fail-fast如何实现：通过modCount（修改次数）实现。创建迭代器时，会为exceptedModCount赋值int exceptedModCount = modCount，用来记录当前修改次数。迭代器使用next()方法时，会调用checkForComodification()方法，在该方法中检查modCount与exceptedModCount是否相等。

modCount何时会改变：对集合结构进行修改（添加或删除，修改值不算）时，modCount会+1。使用集合本身的方法或迭代器的方法对集合结构进行修改，modCount都会发生改变，而只有使用迭代器方法对集合结构进行修改，exceptedModCount才会发生改变。

如果想避免fail-fast，可以使用CopyOnWriteArrayList代替Arraylist，其使用了fail-safe机制。

fail-safe

什么是safe-fast：安全失败机制。使用 java.util.concurrent.ConcurrentHashMap，无论改变值还是改变结构，都不会抛出 ConcurrentModificationException。

fail-safe如何实现：在原集合的 copy 上遍历。存在两个问题：创建 copy 需要额外的空间和时间上的开销；不能保证遍历的是最新的内容。
